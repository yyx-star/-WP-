# 题后总结：

###  （一）本地文件包含（Local File Inclusion, LFI）

LFI 是一种 Web 安全漏洞，攻击者通过操纵应用程序的文件包含功能，让服务器**包含并执行**本应无权访问的本地文件。

#### 1. 产生原因

PHP 中常用的文件包含函数：

- `include()`
- `require()`
- `include_once()`
- `require_once()`

当开发者**直接将用户输入传递给这些函数**，且没有充分过滤时，就产生 LFI 漏洞。

例如：

```php
// 危险代码
$page = $_GET['page'];
include($page . ".php");
```

那么可以传入

```
?page=../../../../etc/passwd%00
```

从而读取系统敏感文件。

过程：

第一步：URL 解码

```
../../../../etc/passwd%00
↓ URL 解码
../../../../etc/passwd\0
```

第二步：PHP 字符串拼接

```php
$page = "../../../../etc/passwd\0";
include($page . ".php");
```

拼接后变成：

```
"../../../../etc/passwd\0.php"
```

那么就可以去读敏感文件了

#### 2.LFI 通用测试流程

1. 找参数    → 题目提示 language
2. 直接测试  → ?language=/etc/passwd
3. 如果失败  → 尝试路径遍历 ../../../../etc/passwd
4. 如果还失败 → 尝试空字节截断 %00 或伪协议
5. 成功读取  → 找 flag
