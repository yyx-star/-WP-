# 题后总结

## （一）CSRF

CSRF，全名 Cross Site Request Forgery，跨站请求伪造。很容易将它与 XSS 混淆，对于 CSRF，其两个关键点是跨站点的请求与请求的伪造，由于目标站无 token 或 referer 防御，导致用户的敏感操作的每一个参数都可以被攻击者获知，攻击者即可以伪造一个完全一样的请求以用户的身份达到恶意目的。

#### 1.常见触发点

- 通过 URL 加载远程图片、文件、网页截图
- 从远程地址获取 XML/JSON 数据
- 网页抓取、RSS 订阅、翻译服务
- 在线文档预览、PDF 生成

#### 



#### 2.CSRF 类型 

按请求类型，可分为 GET 型和 POST 型。

按攻击方式，可分为 HTML CSRF、JSON HiJacking、Flash CSRF 等。



#### HTML CSRF

利用 HTML 元素发出 CSRF 请求，这是最常见的 CSRF 攻击。

HTML 中能设置 `src/href` 等链接地址的标签都可以发起一个 GET 请求，如：

```
<link href="">
<img src="">
<img lowsrc="">
<img dynsrc="">
<meta http-equiv="refresh" content="0; url=">
<iframe src="">
<frame src="">
<script src=""></script>
<bgsound src=""></bgsound>
<embed src=""></bgsound>
<video src=""></video>
<audio src=""></audio>
<a href=""></a>
<table background=""></table>
......
```

还有 CSS 样式中的：

```
@import ""
background:url("")
......
```

也可使用表单来对 POST 型的请求进行伪造。

```
<form action="http://www.a.com/register" id="register" method="post">
  <input type=text name="username" value="" />
  <input type=password name="password" value="" />
</form>
<script>
  var f = document.getElementById("register");
  f.inputs[0].value = "test";
  f.inputs[1].value = "passwd";
  f.submit();
</script>
```

#### Flash CSRF

Flash 也有各种方式可以发起网络请求，包括 POST。

```
import flash.net.URLRequest;
import flash.system.Security;
var url = new URLRequest("http://target/page");
var param = new URLVariables();
param = "test=123";
url.method = "POST";
url.data = param;
sendToURL(url);
stop();
```

Flash 中还可以使用 `getURL`、`loadVars` 等方式发起请求。

```
req = new LoadVars();
req.addRequestHeader("foo", "bar");
req.send("http://target/page?v1=123&v2=222", "_blank", "GET");
```

### 绕过方式

- **短链接重定向**：`http://short.url/xxx` 跳转到内网地址
- **DNS 重绑定**：域名解析先返回正常 IP，后返回内网 IP
- **URL 编码/进制转换**：`http://0x7f000001`、`http://2130706433` 绕过 IP 黑名单
- **利用重定向**：使用允许的域名跳转到内网
- **`file://` 协议**：读取本地文件（若未禁用）
- **`gopher://` 协议**：构造任意 TCP 请求，攻击内网服务





## （二）AJAX 请求详解

**AJAX** = **Asynchronous JavaScript and XML**（异步 JavaScript 和 XML）

它是一种在不重新加载整个网页的情况下，与服务器交换数据并更新部分网页内容的技术。

## （三）DOM 

**DOM** = **Document Object Model**（文档对象模型）

DOM 是浏览器将 HTML 文档解析成的一个**树形结构**，它把网页的每一个部分都表示为一个**对象**，让 JavaScript 可以动态访问和修改网页内容。

DOM 让 JavaScript 可以：

1. **查找**元素
2. **读取**元素属性和内容
3. **修改**元素属性和内容
4. **删除**元素
5. **创建**新元素

#### DOM 中的节点类型

| 节点类型     | 例子                            | 在代码中的体现                                 |
| :----------- | :------------------------------ | :--------------------------------------------- |
| **元素节点** | `<div>`、`<a>`、`<p>`           | `document.getElementById()` 返回的就是元素节点 |
| **属性节点** | `id="ACC"`、`link="http://..."` | `element.attributes.link` 访问属性节点         |
| **文本节点** | "Accounting Department"         | 标签内的文字                                   |
| **文档节点** | `document`                      | 整个 DOM 树的根                                |

#### 绕过方式一：域名解析绕过

python

```
# DNS 重绑定攻击流程
# 1. 攻击者控制一个域名，TTL 设为 0
# 2. 第一次解析返回合法 IP（如 1.2.3.4）
# 3. 服务器校验通过后，第二次解析返回内网 IP（如 127.0.0.1）
# 4. 请求被发送到内网
```

**防御**：重新解析域名并对比 IP，或禁用 DNS 重绑定。

#### 绕过方式二：302 重定向绕过

```
// 攻击步骤：
// 1. 提交 http://evil.com/redirect
// 2. evil.com 返回 302 Location: http://169.254.169.254/
// 3. 如果 CURLOPT_FOLLOWLOCATION 开启，请求被重定向到内网
```

**防御**：`curl_setopt($ch, CURLOPT_FOLLOWLOCATION, false);`

#### 绕过方式三：IPv6 绕过黑名单

```
http://[::1]/                     // localhost IPv6

http://[0:0:0:0:0:0:0:1]/         // IPv6 完整格式
```
